# 完整系统范围

## 目标

老板和获授权店员可在苹果、安卓和电脑上完成二手机入库、查询、出库、盘点、预订、送修和标签打印；老板在店内外都能查看实时经营状态。

## 使用角色

- 老板：查看成本、利润、全部流水，管理店员、模板与设置。
- 店员：入库、查询、出库、盘点、预订、送修；默认看不到收货成本、最低价和总利润。
- 店内打印代理：只读取本店待打印任务并回写结果，不允许浏览经营报表。

## 核心流程

### 入库

1. 上传或截取爱思助手设备详情。
2. 本地OCR读取型号、颜色、系统、电池、充电次数、序列号、IMEI。
3. 识别置信度不足时才调用AI整理字段。
4. 人工补充容量、成色、成本和标价。
5. 后端原子生成库存编号并保存原图。
6. 创建30×40mm打印任务；店内Windows代理打印到B1并回写结果。

### 出库

1. 手机扫描标签，或输入库存编号、IMEI尾号。
2. 显示唯一设备并要求确认成交价和支付方式。
3. 后端在一个事务内校验状态、创建销售记录、变更库存状态并记录日志。
4. 所有在线设备自动刷新经营数据。

### 店外查看

- 网页和数据均部署在云端，电脑关闭时仍可查看与操作库存。
- 打印任务可在店外创建，店内打印代理在线后自动处理。
- 所有访问必须登录；数据库按门店和角色执行行级权限。

## 库存状态

`在库`、`已预订`、`已售待取`、`已售`、`送修中`、`借出检测`、`同行调货`、`退货处理中`、`已报废`。

状态变化必须产生不可覆盖的操作事件。删除设备只允许老板执行软删除，不直接删除历史销售和操作记录。

## Token控制

- 搜索、扫码、编号、统计、权限、打印和状态流转全部使用普通程序，不调用AI。
- OCR优先在本地执行。
- AI只接收单张截图的OCR结果和必要图像，不读取全店库存。
- 同一版式识别规则缓存复用；字段完整时不调用AI。

## 数据安全

- IMEI、序列号和截图只对本店登录用户开放；列表默认脱敏。
- 成本、最低价和利润存放在独立财务表，仅老板可读。
- 云端密钥不写入网页；打印代理密钥只保存在店内电脑。
- 每日导出备份；正式营业环境启用云端自动备份。
