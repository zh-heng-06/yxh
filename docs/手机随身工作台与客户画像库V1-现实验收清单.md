# 手机随身工作台与客户画像库 V1 现实验收清单

版本：V1.1.1（2026-07-20）

测试规则：先用专门的测试手机、测试客户称呼和测试手机号。不要用真实顾客资料做演练。每项在“结果”填写通过/不通过，并记录现象。

## 一、打开与排版

1. 电脑打开 `http://127.0.0.1:4180/`，确认是横屏宽版：首页指标横向排列、库存两列、顶部有“客户中心”。
2. 苹果手机和安卓手机连接店内同一 Wi-Fi，打开电脑显示的局域网地址，确认自动进入竖屏“手机随身工作台”。
3. 手机首页确认六个入口：完整入库、扫码出库、忙时快速入库、查库存、待补全、客户记录。

## 二、完整入库和外观图

4. 在手机“完整入库”的单个文字框点击系统键盘麦克风，一次说出品牌、型号、容量、15位IMEI、收货成本、销售标价及可选机况，点击“识别信息并填入”。

   - 核对识别出的六项核心资料；故意漏说一项，确认页面明确提示缺少内容且不会自动入库。
   - 核对无误后确认入库，确认设备进入可售库存并由电脑打印标签。
   - 打开手机“库存列表”，确认每台设备显示外观完成状态、总照片数、正面/背面/边框/瑕疵照片数，以及颜色、成色、电池、库位和备注摘要。
5. 故意漏一项或输入非15位IMEI，确认系统阻止保存并显示明确原因。
6. 入库成功后点“拍正面、背面和瑕疵图”；分别用苹果和安卓各测一次。
7. 只拍背面就点“无明显瑕疵”，确认系统提示还缺正面。
8. 拍正面和背面后选“无明显瑕疵”，确认首页“待拍外观”数量减少。
9. 另一台设备拍正面、背面和瑕疵图后选“已拍瑕疵”，在电脑详情中逐张打开，确认分类、方向和清晰度正确。
10. 不拍照片直接出售一台测试机，确认系统只提醒、不阻止出库。

## 三、忙时快速入库

11. 只填型号、容量、IMEI和收货成本做快速入库，确认生成库存编号并进入“待补全”。
12. 扫这台测试机出库，确认系统阻止出售。
13. 在手机“待补全”点开该机，补销售标价、成色、电池和库位，保存后再次扫码，确认可以出售。

## 四、成交速记和客户待办

14. 扫码出库，在“成交/客户速记”用手机键盘麦克风说一句话；确认网页没有单独申请麦克风权限，最后只保存文字。
15. 出库完成后确认“客户记录”的待补数量增加1，重复刷新不会增加第二条。
16. 打开待补记录，点“本地整理建议”，确认预算、意向型号、关注点、来源、未成交原因或跟进时间只是建议，必须人工确认后保存。
17. 输入“15岁学生、家庭、收入、住址”等测试文字，确认系统提醒删除不必要的敏感信息。
18. 对一条待办点“无需补充”，确认从待补列表消失；另一条补齐后保存，确认也从待补列表消失。

## 五、匿名与实名客户

19. 新增一条不填称呼和手机号的到店记录，确认可正常保存为匿名互动。
20. 填称呼或手机号但不勾选客户同意，确认系统拒绝保存；勾选后才能建立客户档案。
21. 用同一测试手机号记录两次不同互动，确认客户搜索只有一个档案，历史中有两次互动。
22. 用店员账号打开客户列表，确认手机号遮挡；进入单个客户后点查看，确认完整号码只在本次详情显示。
23. 老板到“设置 → 系统操作日志”，确认存在“查看客户电话”日志，日志中没有完整手机号和原始速记。
24. 给测试客户设置下一次跟进时间，时间到后确认首页和客户中心出现待跟进提醒。
25. 用测试客户执行“确认匿名化”，确认姓名、电话和原始文字消失，但销售账目和匿名统计仍保留。

## 六、经营汇总、权限和恢复

26. 老板打开电脑“客户中心”，确认可看来源、意向型号、关注点和未成交原因汇总；店员看不到经营洞察。
27. 确认系统没有客户资料批量导出按钮或接口；库存和销售导出仍正常。
28. 关闭手机网页并重新打开、重启电脑服务后复查库存、照片、待办和客户记录仍存在。
29. 在“设置”执行一次立即备份，确认备份显示校验通过。
30. 仅在测试库执行恢复演练，确认恢复后库存、销售、交接卡、账号和客户资料一致。
