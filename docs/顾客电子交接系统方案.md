# 顾客电子交接系统方案

状态：下一阶段候选，2026-07-18 已确认产品方向，尚未开发。

## 为什么做

掌柜台 V1 已经解决门店内部的入库、标签、出库、账本、质保、售后与追溯。下一步可以把成交和验机结果安全地交给顾客，让顾客扫码后直接在手机中留存，同时为门店留下可持续的售后和复购入口。

第一阶段不做复杂会员系统，先做简单可靠的“电子交接单”。它不能干扰现有出库流程，也不能让顾客进入内部库存系统。

## 使用场景

### 已成交

出库完成后自动生成电子交接单，包含：

- 交接单编号、成交日期、门店名称与联系方式。
- 型号、容量、颜色、IMEI 尾号和库存编号。
- 成交价、支付方式和手机壳、手机膜、充电头、充电器等赠品。
- 门店质保天数、具体到期日期和售后说明。
- 数据备份、账号退出、外观、功能、赠品等交机确认事项。
- 售后查询、再次回收和联系门店入口。

店员向顾客展示二维码，顾客微信扫码查看，并点击“保存交接单”把图片或 PDF 保存到手机。

### 未成交

生成“验机与报价单”，包含型号、容量、电池、成色、本次报价、影响价格的原因、报价有效期和再次联系入口。即使当天没有成交，顾客手机里仍能留下门店资料。

### 回收、维修与售后

- 回收成交：回收设备、价格、时间和双方确认信息。
- 维修接机：故障描述、外观照片、预计费用、接机时间和进度查询。
- 售后受理：问题、受理时间、处理进度、最终结果和取机信息。

## V1 建议流程

1. 店员按现有流程完成出库。
2. 系统自动建立一条顾客交接记录，并冻结当时的成交信息快照。
3. 页面显示顾客二维码。
4. 顾客扫码核对，不需要注册账号。
5. 顾客下载一张适合微信保存的长图；需要时也可下载 PDF。
6. 店内设备详情保留“再次展示、补发、作废和查看记录”入口。

未成交报价单从验机结果页单独生成，不应伪造成交记录。

## 当前局域网限制

当前服务器使用店内 `192.168.x.x` 局域网地址。顾客离开店内 Wi-Fi 后，网页链接无法继续打开。因此第一版必须保证：

- 顾客当场可以把交接单长图或 PDF 真正下载到手机，而不是只收藏局域网网址。
- 店内数据库永久保留交接快照，之后可以重新展示或补发。
- 将来部署公网服务后，再提供长期可访问的顾客专属页面、质保提醒和维修进度。

## 安全和隐私边界

- 顾客页面与内部库存页面完全隔离，不复用老板或店员登录状态。
- 顾客只能看到本次交接需要的信息；不显示回收成本、利润、完整 IMEI、其他库存或其他顾客资料。
- 二维码使用高强度随机令牌，不能通过修改连续编号查看别人的单据。
- 顾客手机号不是第一版必填项；需要保存前必须明确说明用途。
- 每次查看、下载、补发和作废均写入操作日志。
- 交接快照只能更正并保留历史，不能静默覆盖原成交信息。
- 真实顾客资料、交接单文件和访问令牌不得提交 GitHub。

## 后续可增加的服务

- 质保到期提醒和售后进度查询。
- 维修进度、报价确认和取机通知。
- 一键再次回收、换机提醒和老顾客转介绍码。
- 数据迁移与账号退出检查单。
- 电子保修卡、门店导航和联系微信。
- 顾客授权后的服务评价与复购记录。

## 开发顺序

1. 成交电子交接单：快照、二维码、手机查看、长图下载、补发、作废。
2. 未成交验机报价单：报价有效期、原因说明和再次联系。
3. 维修/售后交接：进度与取机确认。
4. 公网顾客页：完成域名、HTTPS、隐私告知、数据备份后再开放。
5. 提醒、转介绍和轻量会员功能。

## 第一阶段验收标准

- 已成交设备一键生成唯一交接单，成交快照与每日账本一致。
- 苹果和安卓微信均能扫码打开并保存长图。
- 顾客离开 Wi-Fi 后仍能在相册查看已下载交接单。
- 顾客页面不包含成本、利润或完整 IMEI。
- 二维码随机且无法枚举；作废后不能再次查看。
- 补发和作废均有老板可查的操作日志。
- 交接单功能失败时不影响原有出库成功和每日账本。
