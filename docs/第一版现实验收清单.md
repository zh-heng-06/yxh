# 掌柜台第一版现实验收清单

版本：V1（局域网正式使用版）

测试规则：先用测试手机完成 01–16；全部通过后再录正式库存。打印、扫码和断电只能依靠真实设备验收，自动测试无法替代。

## 开店前必须通过

- [ ] **01 启动与登录**：双击“启动局域网系统.cmd”，电脑和手机都能登录，刷新页面不白屏。
- [ ] **02 数据库与磁盘**：设置页显示数据库正常、磁盘空间正常。
- [ ] **03 备份**：设置页显示“每天19:00自动备份”，点击立即备份后显示校验通过。
- [ ] **04 手机连接**：手机与电脑同一 Wi-Fi，扫描“手机连接”二维码后能打开系统。
- [ ] **05 权限**：老板能看成本和利润；店员看不到成本、利润、备份恢复和系统操作日志。

## 每天营业主流程

- [ ] **06 苹果直连/截图入库**：优先数据线连接 iPhone，在手机上点“信任”，核对自动读取字段后补价格入库；直连异常时用爱思截图入库。
- [ ] **07 安卓/华为直连入库**：开启开发者选项和 USB 调试，连接后在手机上允许调试；系统读出品牌、型号、系统和容量，未开放的 IMEI/电池字段留空并明确提示，人工核对补齐后成功入库。至少各测试一台华为和一台其他安卓机。
- [ ] **08 重复保护**：再次录入相同 IMEI，系统必须拒绝。
- [ ] **09 标签实物**：入库后只打印一张 40×30 横向标签；无边框、内容不跨纸、二维码清楚。
- [ ] **10 已打印隐藏**：打印成功后刷新页面，“打印标签”按钮不再出现。
- [ ] **11 苹果扫码出库**：手机扫码后先显示设备资料；核对无误，填写成交价、支付、赠品和质保天数，再确认出库。
- [ ] **12 安卓扫码出库**：安卓手机完成与 11 相同流程。
- [ ] **13 重复出库保护**：再次扫描已售标签，系统必须阻止再次出库。
- [ ] **14 每日账本**：型号、容量、串号、赠品、成交价、回收价和利润与刚才交易一致。

## 异常与追溯

- [ ] **15 库存提醒**：有超30/60/90天库存、到期预订、久未取机、久未修好或待处理售后时，首页出现对应提醒并能点开。
- [ ] **16 售后闭环**：打开已售设备，能看到质保到期日；登记售后问题、填写处理结果和成本后，状态变为已完成。
- [ ] **17 操作日志**：设置页能看到登录、入库、出库、退货、账目更正、打印失败、售后和备份恢复记录。
- [ ] **18 退货再售**：测试机退货后恢复在库，再次销售成功；两次销售和一次退货记录都保留。
- [ ] **19 重启保存**：关闭服务后重新启动，库存、账本、售后和日志仍存在。
- [ ] **20 断电恢复**：在没有写入操作时正常关机再开机，数据库自检正常；随后立即做一次备份。

任何 01–14 项失败，都先不要把它当作唯一账本使用。把“编号 + 页面截图 + 实体标签照片”发给开发者定位。
