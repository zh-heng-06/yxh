# 掌柜台现实功能测试清单

使用方法：严格按编号测试。每项成功打 `✓`，失败打 `×` 并拍照记录。P0 项有任何一项失败，先停止营业使用并反馈；涉及退货、恢复备份的项目只使用测试手机。

## P0：先确认系统可靠（必须全部通过）

- [ ] **01 电脑启动**：双击根目录的“启动局域网系统.cmd”。预期：浏览器打开掌柜台登录页，没有白屏。
- [ ] **02 正式账号登录**：用老板账号登录。预期：看到店名、手机库存、入库等按钮。
- [ ] **03 数据库自检**：打开“设置”。预期：运行状态显示数据库正常。
- [ ] **04 打印机状态**：连接并开启 NIIMBOT B1，再打开“设置”。预期：打印机显示已连接，并能看到端口。
- [ ] **05 手机连接**：电脑点“手机连接”，苹果和安卓分别扫码。预期：同一 Wi-Fi 下都能打开登录页。
- [ ] **06 手机登录**：两台手机分别登录。预期：库存数量、数据与电脑一致。
- [ ] **07 刷新与重启**：刷新网页，再关闭服务窗口并重新启动。预期：已有库存没有丢失，页面不白屏。
- [ ] **08 手工备份**：设置中点“立即备份数据库”。预期：备份列表出现带当前时间的新文件。
- [ ] **09 密码修改**：设置中修改一个测试店员密码，退出后用新密码登录。预期：旧密码失效，新密码有效。
- [ ] **10 权限隔离**：老板创建测试店员并登录。预期：店员看不到收货成本、利润、备份恢复和员工管理。

## P1：入库、标签、扫码出库（营业主流程）

- [ ] **11 苹果手机截图入库**：上传一张爱思助手“设备详情”完整截图。预期：型号、颜色、系统、电池、充电次数、IMEI、序列号自动识别。
- [ ] **12 苹果手机人工确认**：补充容量、成本、标价，检查识别值后确认。预期：生成唯一库存编号，库存新增 1 台。
- [ ] **13 安卓手机截图入库**：上传一张安卓验机/设备信息截图。预期：能识别常见字段；未识别字段可直接补填，不阻塞入库。
- [ ] **14 输入校验**：故意输入少于 15 位 IMEI、负数价格。预期：系统阻止提交并说明原因。
- [ ] **15 低于成本提醒**：标价填写得低于成本。预期：系统要求再次确认，避免误售。
- [ ] **16 入库自动打印**：勾选“入库成功后立即打印”。预期：只出 1 张标签，没有浏览器打印框。
- [ ] **17 标签实物排版**：查看 40×30 横向标签。预期：无边框、中文不乱码、六行紧凑、内容均在一张纸内、二维码清楚。
- [ ] **18 连续打印**：对同一测试机补打 10 张。预期：不跳纸、不分成两张、不越界，位置基本一致。
- [ ] **19 标签二维码**：用手机拍摄标签二维码执行扫码出库。预期：准确找到对应库存，不会找到别的机器。
- [ ] **20 编号/IMEI 出库**：分别用完整库存编号、完整 IMEI、IMEI 尾号查找。预期：唯一结果可进入出库；多结果时不能误出库。
- [ ] **21 出库成交**：填写实售价和支付方式确认。预期：库存状态变为已售，当日销量、销售额、利润更新。
- [ ] **22 重复出库保护**：再次扫已售标签。预期：系统阻止重复销售，并明确显示当前状态。
- [ ] **23 详情编辑与补打**：修改颜色/系统/标价，保存并补打。预期：详情和新标签同步更新，旧操作记录保留。
- [ ] **24 原始凭证**：打开设备详情。预期：能看到入库原始截图、实拍照片和完整操作时间线。

## P2：库存状态与异常流程

- [ ] **25 预订**：把在库手机设为预订，填写客户和到期时间。预期：库存显示已预订，普通出库前有明确状态提示。
- [ ] **26 取消预订**：取消上述预订。预期：恢复在库，预订和取消记录均保留。
- [ ] **27 送修**：填写故障、维修商、预计费用后送修。预期：状态变为送修中，不计入可售库存。
- [ ] **28 维修完成**：填写实际费用和结果。预期：恢复在库，维修前后信息留在时间线。
- [ ] **29 销售退货**：对测试成交执行退货并重新入库。预期：状态恢复在库，退款记录存在，当日报表冲减原销售收入和利润。
- [ ] **30 退货后再售**：将退回的同一设备再次销售。预期：允许第二次销售，两次销售和一次退货记录均存在。
- [ ] **31 其他状态**：依次测试已售待取、借出测试、同行调拨、退货处理中、报废。预期：列表筛选与详情状态一致，不能把不可售状态误当在库。
- [ ] **32 库位**：给三台测试机设置 A柜/B柜。预期：详情可修改、搜索和盘点时库位正确。

## P2：手机盘点

- [ ] **33 开始盘点**：选择 A柜开始。预期：显示应盘数量和已扫数量。
- [ ] **34 连续扫码**：连续拍摄/输入三台标签。预期：每台只增加一次，手机有成功反馈。
- [ ] **35 重复扫描**：同一标签扫两次。预期：明确提示重复，不重复计数。
- [ ] **36 错区设备**：在 A柜盘点中扫描 B柜设备。预期：提示放错区域。
- [ ] **37 完成盘点**：故意漏扫一台后完成。预期：列出漏扫库存编号、型号和库位，盘点记录可追溯。

## P3：报表、导入与数据安全

- [ ] **38 经营报表**：选择今天生成报表。预期：销量、销售额、利润与实际测试单一致，退货已正确冲减。
- [ ] **39 库龄与绩效**：检查库龄、机型销量、店员业绩。预期：分组和数量正确，店员看不到老板利润数据。
- [ ] **40 导出库存**：导出库存 CSV，用 Excel 打开。预期：中文正常、每台一行、编号/IMEI 不被科学计数法破坏。
- [ ] **41 导出销售**：导出销售 CSV。预期：成交、退货相关数据与时间正确。
- [ ] **42 批量导入**：下载模板，填 2 台虚拟手机后导入。预期：提示成功数量；重复 IMEI 被拒绝且不产生脏数据。
- [ ] **43 备份恢复演练**：先立即备份，再新增一台“恢复测试机”，恢复刚才备份。预期：恢复测试机消失，备份前数据完整。此项不要在真实营业录单后执行。
- [ ] **44 电脑断电模拟**：没有操作写入时正常关机再开机。预期：系统随开机启动或可一键启动，数据库自检正常，库存不丢。

## P4：本地智能提效（不消耗云端 AI token）

- [ ] **45 语音快速入库**：说“苹果16 Pro，256G，黑色，成本六千，卖六千八”。预期：识别后带入入库表；提交前仍需人工确认。
- [ ] **46 文字快速入库**：输入同类自然语言。预期：本地解析型号、容量、颜色、成本和标价。
- [ ] **47 智能定价建议**：对有历史成交的机型点“价格建议”。预期：展示参考区间和依据，不自动改价。
- [ ] **48 销售文案**：点“销售文案”。预期：生成可复制描述，不包含收货成本和完整 IMEI。
- [ ] **49 今日经营总结**：打开智能助手。预期：显示当天入库、销售、库存积压或异常提醒。
- [ ] **50 瑕疵照片**：上传正面、背面和瑕疵照片并填写说明。预期：照片永久保存在该库存详情下，不与别的手机混淆。

## P2：行情中心与独立定价

- [ ] **51 老板权限**：老板登录后能看到“行情中心”；测试店员登录后看不到入口，直接访问接口也被拒绝。
- [ ] **52 录入回收行情**：按实际看到的公众号/服务号报价填写来源、回收报价、型号、容量、成色、电池、拆修情况和日期。预期：保存后立即出现在匹配行情中。
- [ ] **53 录入销售行情**：录入京东、抖音或其他合规来源中相同条件的销售报价。预期：与回收报价分开统计，不混成一个价格。
- [ ] **54 条件匹配**：分别改变容量、成色、电池和拆修情况查询。预期：只使用匹配或未标明条件的证据，不把明显不同配置混入。
- [ ] **55 内外数据独立**：查询店内有成交记录的型号。预期：页面分别显示外部回收/销售样本和门店成交/库存样本，并显示各自数量与中位数。
- [ ] **56 参考区间**：检查建议回收区间、建议销售区间、预计最低毛利、依据和可信度。预期：样本不足时明确显示证据不足；系统不会自动修改库存标价。
- [ ] **57 老板最终定价**：填写最终回收价、最终销售价和调整原因。预期：作为一条独立决策保存，历史行情本身不被改写。
- [ ] **58 错误行情纠正**：录入一条虚假测试报价后删除。预期：仅删除选中的行情，重新查询后区间会更新，老板历史决策仍保留。
- [ ] **59 完整报价表识别**：在行情中心填写“博能二手回收”，粘贴当天完整报价图片链接后点击“下载并本地识别”。预期：约一分钟后显示报价日期、识别行数和可横向查看的价格表；过程不调用云端 AI。
- [ ] **60 关键价格核对**：用 2026-07-18 样本核对 iPhone 16 Pro Max 256GB。预期：靓机 5930、小花 5750、大花 4850、外爆 4700、内爆可测 3700；今后新图片以原图为准逐行抽查。
- [ ] **61 人工修改与跳过**：修改一个故意填错的数字，并取消勾选一行。预期：修改值被导入，未勾选行完全不写入。
- [ ] **62 批量导入与防重复**：确认批量导入后，再导入同一张表。预期：第一次新增行情；第二次提示跳过同日、同来源、同型号、同容量、同档位、同价格的重复记录。
- [ ] **63 成色自动对应**：查询 iPhone 16 Pro Max 256GB 的“95新”。预期：自动使用外部报价表“靓机”档；查询“9成新”使用“小花”档；页面也允许直接选择报价表原始档位。
- [ ] **64 来源与原图留档**：查看导入结果和本机数据目录。预期：行情保留来源、报价日期和原图片链接，成功识别的原图保存在本机，方便以后追溯。
- [ ] **65 官网自动行情状态**：进入行情中心查看“官网每日自动行情”。预期：显示 5041 苹果有保、5042 苹果无保两个公开页面、每天14:30计划和最近一次同步结果。
- [ ] **66 无保完整度**：点击 5042“立即同步”。预期：2026-07-18 样本显示 101/101 行、505 条价格；iPhone 16 Pro Max 512GB 为 6630、6430、5450、5300、4400。
- [ ] **67 有保六档行情**：点击 5041“立即同步”。预期：2026-07-18 样本显示 43/43 行、245 条价格；“高保充新”有报价时独立保存，原表空白时保持空白，不伪造数字。
- [ ] **68 不完整拒绝保护**：使用裁剪图、模糊图或人为缺行测试图。预期：显示“校验失败，未导入”，缺失数据不会进入行情查询。
- [ ] **69 定时与幂等**：电脑在14:30保持开机联网，等待自动任务；之后对同一原图再次点击同步。预期：自动任务成功留档；重复操作显示今日已经导入，不再次OCR、不产生重复行情。
