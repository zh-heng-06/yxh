# 掌柜台 · 手机门店完整系统

在另一台电脑继续开发时，请先阅读 [`docs/异地继续开发交接.md`](docs/异地继续开发交接.md)。该文档包含当前完成状态、聊天中已经确定的产品原则、未完成功能以及数据隐私边界。

这是正式系统目录。`D:\Codex\phone-shop-prototype` 只作为交互原型和设备联调参考，不作为生产数据源。

## 正式系统组成

- `apps/web`：苹果、安卓和电脑共同使用的PWA。
- `apps/print-agent`：店内Windows打印代理，接收云端任务并驱动 NIIMBOT B1。
- `supabase/migrations`：云端数据库、登录权限、操作日志和打印队列。
- `docs`：需求、验收标准、安全与部署文档。

## 局域网启动

首次在新电脑部署时，先安装 Python 3.12，然后在项目目录执行：

```powershell
python -m pip install -r requirements.txt
```

如需使用 NIIMBOT B1 标签打印，在 Windows 上运行 `apps/print-agent/build.cmd` 生成打印组件。

双击 `启动局域网系统.cmd`。第一次打开会要求创建门店名称、老板用户名和密码。手机与电脑连接同一个Wi-Fi后，点击电脑页面的“手机连接”扫码进入。

正式本地数据库位于 `apps/local-server/data/store.db`，采用SQLite WAL模式。不要直接编辑数据库文件。

入库默认使用“爱思截图入库”：选择爱思助手的设备详情截图后，电脑本地OCR会自动识别型号、颜色、系统版本、电池健康、充电次数、IMEI和序列号。由于爱思截图不含容量、收货成本和销售价，这三项补充后即可生成库存编号并入库。截图不会上传互联网，识别过程不调用云端AI。

入库页面默认勾选“入库成功后立即打印40×30横向标签”。正式系统使用本地 `apps/print-agent/LabelPrinter.exe` 把中文和设备信息先渲染成203dpi黑白位图，再通过 Windows 队列发送给 `NIIMBOT B1`，避免浏览器打印造成乱码。未成功打印的库存卡片会显示“打印标签”；只要成功打印过一次，首页和详情便不再显示打印按钮，避免重复打印。

## 当前建设状态（局域网版）

- 已完成 SQLite 账号、老板/店员权限、数据库自检、每日/手工备份、恢复、密码修改和开机启动。
- 已完成苹果截图本地 OCR 入库、安卓常见字段兼容、手工补录、唯一库存编号和原始截图留档。
- 已完成 40×30mm 横向无边框标签、库存二维码、入库自动打印和打印记录。
- 已完成手机扫码/编号/IMEI 出库、详情编辑、预订、送修、退货再售、库位和多种库存状态。
- 首页默认展示今日入库，并提供今日出库、预订中、已售待取、待打印、超30天等重点卡片；“全部库存”保留在筛选末尾，搜索始终覆盖全部库存。

- 旧的本地规则“智能助手”界面已经移除。后续智能助手将接入 Codex，按明确授权控制电脑完成工作。
